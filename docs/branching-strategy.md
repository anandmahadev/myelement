# Branching Strategy

## Main Branch
- `main` is always deployable.

## Branch Names
- `feat/<scope>` for feature work
- `fix/<scope>` for bug fixes
- `docs/<scope>` for documentation-only changes

## Merge Rules
- Rebase or merge latest `main` before PR.
- Squash noisy fixup commits before merge.
- Keep PRs small and purpose-specific.
