# Contributing to JoByte

## Workflow
1. Create a feature branch from `main`.
2. Keep commits focused and atomic.
3. Run frontend checks before opening a PR.
4. Open a PR with clear screenshots for UI changes.

## Commit Message Convention
Use short imperative messages:
- `feat: add employer trend chart`
- `fix: correct auth role toggle`
- `docs: add API onboarding example`

## Pull Request Checklist
- [ ] Scope is focused and reviewable
- [ ] No secrets in committed files
- [ ] API contract changes reflected in `API_SPEC.md`
- [ ] Manual test evidence included
